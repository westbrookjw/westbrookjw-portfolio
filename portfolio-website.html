AWS S3 + CloudFront Static Website Hosting

End-to-end cloud-based deployment using AWS and Cloudflare

Project Overview

Designed and implemented a static website hosting solution using Amazon S3 and CloudFront. The process involved provisioning an S3 bucket for static file storage, enabling static website hosting, configuring bucket permissions securely via Origin Access Control (OAC), and deploying a CloudFront distribution for global content delivery with low latency and HTTPS support. The setup also included optional DNS integration via Cloudflare for custom domain routing.

Architecture

  • S3 bucket configured for static website hosting
  • CloudFront distribution pointing to the S3 static site endpoint
  • SSL/TLS enabled via ACM and integrated with CloudFront
  • Cloudflare used for DNS and global CDN routing

S3 Bucket Setup Steps

How to setup an S3 Bucket

  1. Create AWS account.
  2. View all services → Select S3 → Create bucket.
  3. Name the bucket (e.g., your-site-name) → leave the rest as default unless you need versioning or encryption.
  4. Go to the bucket’s “Properties” tab.
    • Scroll to “Static website hosting”
    • Enable it
    • Specify index.html as the home page
    • Optionally set error.html for error documents
  5. Go to CloudFront → Distributions → Create Distribution.
  6. In the Origin settings:
    • Choose “Browse S3”
    • Select the bucket you just created
    • Click “Next”
  7. In the Distribution Settings:
    • Leave most defaults
    • Optionally set a custom domain (CNAME) if you have one
    • Note: If using a domain, you’ll later need to update DNS records in your DNS provider (e.g. Cloudflare)
  8. Once the distribution is created, go to “Origins and Origin Groups” tab
    • Select your origin
    • Click “Edit”
  9. Scroll to “Origin access control settings”
    • Ensure Origin access control (OAC) is enabled
    • Copy the S3 bucket policy snippet provided
  10. Go back to S3 → Your bucket → Permissions tab
    • Click “Bucket policy”
    • Paste the policy you copied from CloudFront
    • Save
  11. In S3 Permissions, ensure Block all public access is OFF.
  12. In the Bucket Policy, make sure it allows only CloudFront (via the OAC) to access the content.
  13. Wait for CloudFront to deploy (~10-15 min).
  14. Access the distribution’s domain name (e.g., d1234.cloudfront.net) to verify your static website is working.
  15. (Optional) Set up a custom domain with HTTPS:
    • In Route 53 or Cloudflare, point your domain to your CloudFront distribution using a CNAME
    • Add the custom domain in your CloudFront settings
    • Request a free certificate in AWS Certificate Manager
    • Attach the certificate to your distribution
    • Deploy changes and wait for propagation
  16. Leave all as is and click "Copy policy".
