S3 Bucket Static Website Deployment

End-to-end cloud-based deployment using AWS and Cloudflare

Project Overview

This project showcases a professional static portfolio website deployed using AWS S3 for hosting, CloudFront for global content delivery and HTTPS, and Cloudflare for DNS management and performance enhancement.

Architecture

  • S3 bucket configured for static website hosting
  • CloudFront distribution pointing to the S3 static site endpoint
  • SSL/TLS enabled via ACM and integrated with CloudFront
  • Cloudflare used for DNS and global CDN routing

S3 Bucket Setup

How to setup an S3 Bucket

  1. Step 1: Create AWS account.
  2. Step 2: View all services. Select S3. Create bucket.
  3. Step 3: Name bucket and leave the rest as is.
  4. Step 4: Go to Properties. Enable Static website hosting. Specifiy the home page of the website. Leave the rest as is.
  5. Step 1: Go to Permissions. Add the following code to your Bucket policy:
    6. {
								"Version": "2008-10-17",
								"Id": "PolicyForCloudFrontPrivateContent",
								"Statement": [
									{
										"Sid": "AllowCloudFrontServicePrincipal",
										"Effect": "Allow",
										"Principal": {
											"Service": "cloudfront.amazonaws.com"
										},
										"Action": "s3:GetObject",
										"Resource": "arn:aws:s3:::westbrookjw-portfolio/*",
										"Condition": {
											"StringEquals": {
												"AWS:SourceArn": "arn:aws:cloudfront::your-cloudfront-arn"

									}
